## Decisions

1. **Capability specification frozen as v1** — all future capabilities must follow this structure
2. **Use cases, not services** — each business operation is a single class with one `execute()` method
3. **DTOs for every contract** — `*Request`, `*Response`, `*Summary`, `CommandResult`
4. **`TransactionManager` in core** — shared across all capabilities, not per-capability
5. **camelCase for JSON** — PHP DTOs and TypeScript interfaces use camelCase to match PHP `json_encode` behavior
6. **Route-scoped context** — entity IDs from URL (`?id=N`), never from request body
7. **Skills updated** — PHP Developer and Angular Developer skills aligned around the same capability model
