# Session 005 — Funding Tracker: Complete Module

Date: 2026-07-16

## Goal

Build the complete Funding Tracker module with three data types (Process Tracker, Itemized List, Payments Tracker) — each with table view, import, batch delete, individual CRUD, and company name matching. Both company-level (under `/company/:id/funding-tracker/`) and global admin views (under `/admin/grant-funding/`).

## Decisions
- SEDA Assessment removed — duplicate of Process Tracker
- All three types use generic `INode<T>` + `NodeService` — no custom backend per type
- Company name matching is case-insensitive, trims whitespace
- Unmatched records imported with `company_id: 0` — data never lost
- `AssignCompanyComponent` removed in favor of edit modal company search
- All form components accept `companyId` input for company-level pages
