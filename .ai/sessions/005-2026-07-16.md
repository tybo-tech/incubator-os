# Session 005 — Funding Tracker: Process Tracker, Itemized List, Payments Tracker

Date: 2026-07-16

## Goal

Build the complete Funding Tracker module with three data types (Process Tracker, Itemized List, Payments Tracker) — each with table view, import, batch delete, individual CRUD, and company name matching. ### Key Patterns Established

1. **Company name matching on import** — imports parse company name from column 1, match against `listAllCompanies()` data. Unmatched records get `company_id: 0` and store the name in `data.companyName`. Yellow rows in the table indicate unlinked records.

2. **Select-all + batch delete** — checkbox column in table header, "Delete Selected (N)" button appears when items checked, calls `deleteNodesBatch()`.

3. **Edit modal** — pencil icon opens a shared form component that handles both add and edit with a searchable company dropdown.

4. **Company name resolution** — one `listAllCompanies()` call on load builds both `companyMap` (id→name) and `companyNameToId` (name→id). No individual `getCompanyById()` calls.

## Decisions

- SEDA Assessment was removed — it was a duplicate of Process Tracker with different column names
- All three types use the generic `INode<T>` + `NodeService` pattern — no custom backend per type
- Company name matching is case-insensitive and trims whitespace
- Unmatched records are still imported (with `company_id: 0`) so data is never lost
- The `AssignCompanyComponent` can be reused on any node type to link a company
