# Engineering Standards

## Rule of Three

Every completed task must update exactly three things:

1. Code
2. Session
3. Sprint

Nothing else. This keeps documentation synchronized with implementation.

## Session Discipline

- Every development session creates a new file in `.ai/sessions/`
- File naming: `NNN-YYYY-MM-DD.md` (sequence number + date)
- Every session follows the same template structure
- Sessions are mandatory — every skill writes one

## Skill Evolution

- Skills are improved after real work, not designed upfront
- When a pattern emerges, update the relevant SKILL.md
- Don't anticipate every responsibility
